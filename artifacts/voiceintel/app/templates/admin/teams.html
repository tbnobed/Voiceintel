How auto-routing works

When a voicemail arrives, VoiceIntel evaluates each team's routing rules in priority order (lowest number first). The first rule that matches wins, and the voicemail is assigned to that team.

The simplest setup is a unique forwarding address per team — e.g. forward to team1@mail3.opscal.io for the Sales team, then add a Recipient address rule with that value.

Rules also work on the sender, sender domain, transcript keywords, or caller phone number. Agents can only see voicemails routed to teams they belong to (plus unrouted ones); supervisors and admins always see everything.
