Build a working MVP web app called VoiceIntel.

Goal:
Create a voicemail intelligence dashboard that ingests voicemail recordings from email, transcribes them using a LOCAL Whisper model (no cloud APIs), and provides a searchable analytics UI.

Tech stack:
- Python Flask backend
- PostgreSQL database
- HTML/CSS/JavaScript frontend
- Bootstrap or Tailwind UI
- faster-whisper (local transcription)
- FFmpeg for audio conversion
- IMAP for email ingestion
- SQLAlchemy ORM
- APScheduler for background jobs

CRITICAL REQUIREMENT:
- ALL transcription must be done locally using faster-whisper
- DO NOT use OpenAI API or any external transcription service
- Must support GPU acceleration if available (CUDA), fallback to CPU

--------------------------------------------------
1. Email ingestion (IMAP)
--------------------------------------------------
- Connect to IMAP using environment variables:
  IMAP_HOST, IMAP_PORT, IMAP_USERNAME, IMAP_PASSWORD, IMAP_FOLDER
- Poll every 60 seconds
- Fetch unread emails with audio attachments
- Supported formats: wav, mp3, m4a, ogg, aac
- Save attachments to /storage/voicemails
- Store metadata:
  sender, subject, timestamp, message_id, filename
- Prevent duplicates using message_id + filename
- Mark emails as read ONLY after successful processing

--------------------------------------------------
2. Audio processing
--------------------------------------------------
- Use FFmpeg to convert all audio to:
  mono, 16kHz WAV (required for Whisper optimization)
- Store:
  original_path
  converted_path
  duration
  size
  processing_status

--------------------------------------------------
3. LOCAL transcription (faster-whisper)
--------------------------------------------------
- Use faster-whisper Python package
- Load model at startup (configurable):
  options:
    tiny
    base
    small
    medium
    large-v2

- Detect GPU automatically:
  if CUDA available → use GPU
  else → fallback to CPU

- Transcribe audio files
- Store:
  transcript text
  segments (timestamps optional)
  detected language
  processing time
  errors if any

- Wrap transcription in a service class:
  TranscriptionService.transcribe(file_path)

--------------------------------------------------
4. Keyword + NLP extraction (LOCAL)
--------------------------------------------------
- DO NOT use external APIs
- Use local Python NLP tools:
  - spaCy OR
  - simple keyword extraction (TF-IDF)

Extract:
- keywords
- sentiment (use TextBlob or simple rule-based)
- urgency detection (keyword-based)
- category classification (rule-based for MVP)

Default categories:
- Prayer Request
- Donation Issue
- Technical Issue
- Complaint
- General Inquiry
- Urgent
